# GEMINI.md - Plant-Protein-BERT

## Project Overview
**Plant-Protein-BERT** is a research project focused on training domain-specific BERT-based protein language models exclusively on **Viridiplantae** (green plant) sequences. The project benchmarks these models against general-purpose ESM-2 models to test the hypothesis that domain-specific pretraining outperforms general models on plant-specific downstream tasks.

### Core Stack
- **Frameworks:** PyTorch, PyTorch Lightning
- **NLP/Protein Libraries:** HuggingFace Transformers, Biopython, Tokenizers
- **Configuration:** Hydra (composing experiments and CLI overrides)
- **Experiment Tracking:** Weights & Biases (WandB)
- **Data Handling:** h5py (HDF5 for O(1) random access to 20M sequences), NumPy

## Architecture & Design
- **Models:**
  - `PlantProteinBERT`: A wrapper around HuggingFace `BertForMaskedLM` with parameter-matched configurations for ESM-2 sizes (8M, 35M, 150M).
  - `ESM2FineTuner`: A drop-in replacement for fine-tuning baseline experiments using ESM-2 weights from HuggingFace.
- **Data Pipeline:**
  - `UniProtRecord`: Dataclass for streaming DAT.gz parsing.
  - `StreamingProteinDataset`: HDF5-backed dataset for efficient training on massive FASTA files.
  - `UniProtDataModule`: Lightning DataModule handling splits and lazy loading.
- **Training:**
  - `MLMPretrainer`: LightningModule for Masked Language Modeling (MLM).
  - `FineTuner`: LightningModule for downstream task fine-tuning (e.g., GO term prediction).

## Building and Running

### Setup
```bash
# Install in editable mode with development dependencies
pip install -e ".[dev]"

# Optional: Exact conda environment recreation
conda env create -f environment.yml
```

### Key Pipeline Commands
- **Data Preprocessing:**
  ```bash
  python scripts/parse_uniprot.py data=trembl_full
  ```
- **Pretraining:**
  ```bash
  # Run a specific experiment config
  python scripts/pretrain.py +experiment=pretrain_plant_8m
  
  # Run with CLI overrides
  python scripts/pretrain.py +experiment=pretrain_plant_8m training.optimizer.lr=2e-4
  ```
- **Fine-tuning:**
  ```bash
  python scripts/finetune.py +experiment=finetune_plant_bert_8m \
      training.pretrained_checkpoint=path/to/last.ckpt \
      training.task=go_term_prediction
  ```
- **Evaluation:**
  ```bash
  python scripts/evaluate.py training.pretrained_checkpoint=path/to/best.ckpt
  ```

### Testing
```bash
pytest
```
*Note: Tests use synthetic in-memory data and do not require the full 18GB dataset.*

## Development Conventions

### Coding Style & Standards
- **Tooling:** Uses `Ruff` for linting/formatting and `MyPy` for type checking (configured in `pyproject.toml`).
- **Typing:** Strict type hinting is encouraged throughout the codebase.
- **Configuration:** Always use Hydra `DictConfig` for experiment settings. New model variants or datasets should be added as YAML files in `configs/`.

### Directory Structure
- `src/plant_bert/`: Core package containing models, data logic, and training modules.
- `scripts/`: Entry points for pipeline stages, typically using `@hydra.main`.
- `configs/`: Hierarchical Hydra configurations (model, data, training, experiment).
- `tests/`: Pytest suite using synthetic fixtures.
- `outputs/`: Automatically generated by Hydra, structured as `YYYY-MM-DD/HH-MM-SS/`.

### Contribution Guidelines
- **Adding a new model:** Create a YAML in `configs/model/` and ensure it can be instantiated via `hydra.utils.instantiate`.
- **Modifying Data Loading:** Update `src/plant_bert/data/preprocessing.py` and ensure `StreamingProteinDataset` remains compatible.
- **Logging:** Use the provided `setup_logger` in `src/plant_bert/utils/logging.py` which integrates with WandB.
